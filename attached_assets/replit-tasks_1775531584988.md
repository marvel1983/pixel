## TASK 2 — Metenzi API Integration Service

**Prompt for Replit AI:**

```
Build a server-side Metenzi API client module at /lib/metenzi.ts in TypeScript.

The Metenzi API base URL is: https://metenzi.com/api/public
Authentication: Bearer token via Authorization header.
Write operations (POST, PATCH, DELETE) require HMAC-SHA256 signing.

HMAC signing process:
- Build canonical string: timestamp + "." + method + "." + path + "." + body
- Sign with HMAC-SHA256 using the signing secret
- Include headers: X-Signature (hex) and X-Signature-Timestamp (ms since epoch)

API key and signing secret must be retrieved from the database (encrypted), NOT from hardcoded env vars.

Implement the following functions:

// Products
getProducts(params?: { limit?:number, offset?:number, search?:string, status?:string, category?:string, platform?:string, sortBy?:string, sortOrder?:string, retrieveAll?:boolean }) => Promise<ProductsResponse>
getProductById(id: string) => Promise<Product>
getProductBySku(sku: string) => Promise<Product>

// Orders
createOrder(items: OrderItem[], externalOrderId: string, idempotencyKey: string) => Promise<Order>
getOrderById(id: string) => Promise<OrderWithKeys>
getOrderByExternalId(externalOrderId: string) => Promise<Order>

// Balance
getBalance() => Promise<BalanceData>

// Webhooks
listWebhooks() => Promise<Webhook[]>
createWebhook(url: string, events: string[]) => Promise<WebhookCreated>
deleteWebhook(id: string) => Promise<void>

// Claims
listClaims(status?: string) => Promise<Claim[]>
submitClaim(keyId: string, reason: string, notes?: string) => Promise<Claim>

TypeScript interfaces for all API response types based on:
- Product: { id, sku, name, category, description, shortDescription, platform, b2bPrice, b2bPriceCents, retailPrice, retailPriceCents, currency, stock, status, imageUrl, warrantyDays, instructions, updatedAt }
- Order: { id, number, status, externalOrderId, currency, total, totalCents, createdAt }
- OrderWithKeys extends Order with: { items, keys: [{ code, codeType, status, productId }] }

Include:
- Automatic retry with exponential backoff on 429 (rate limit)
- Error handling with typed errors
- A product sync function that fetches all active products from Metenzi and upserts them into the local PostgreSQL database (run on cron every 30 minutes)
- A background job scheduler (using node-cron) that syncs products automatically
```

## TASK 14 — Metenzi Webhook Handler

**Prompt for Replit AI:**

```
Build the Metenzi webhook handler at /app/api/webhooks/metenzi/route.ts.

Security: verify HMAC-SHA256 signature on every incoming request.
Verify headers: X-Metenzi-Signature, X-Metenzi-Timestamp, X-Metenzi-Event.
Canonical string: timestamp + "." + eventType + "." + rawBody (dot-separated).
Reject events with timestamp older than 5 minutes (replay attack protection).
Use crypto.timingSafeEqual for comparison.
Webhook secret fetched from SiteSettings table (encrypted).

Handle these events:
```
