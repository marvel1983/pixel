Security & Compliance

AES-256-GCM encrypted credential storage with key rotation
CSRF protection, rate limiting, and circuit breakers on every external API
Full audit logging — every credential access, order, and admin action is tracked
Content Security Policy headers, HMAC signature verification, and IP-based access controls
PCI compliance considerations built into the payment and key handling flows
Multi-Tenancy & Role System

Market-based isolation — each market (EUR, Inpoint, Retail) has its own products, pricing, inventory, and customer base
Granular role-based access control (Super Admin, Market Admin, Branch Manager, Customer) with per-market permissions
Branch system allowing sub-locations with their own dashboards and POS
Operational Infrastructure

BullMQ job queues for email, PDF generation, inventory sync, supplier orders, imports, and system tasks — all with retry logic and failure tracking
Redis-backed caching, session management, and rate limiting
Circuit breakers preventing cascade failures when supplier APIs go down
Idempotency service ensuring duplicate orders are never placed
Multi-Supplier Integration

Unified supplier hub connecting to 4+ different supplier APIs (8keys, PrepaidForge, EzCards, Brolexy), each with different auth models
Automated catalog syncing, price intelligence, and repricing engines
Marketplace integrations (Eneba, G2A, Shopify, WooCommerce) with order fulfillment pipelines
Financial Operations

Full ledger system with credit management, settlements, and reconciliation
Payment gateway integration (Checkout.com/Stripe) with webhook verification
Customer invoicing, statements, and payment audit trail with outbox pattern for reliability
Observability

Structured JSON logging with request tracing
Prometheus metrics for API latency, error rates, and queue health
Health checks on database, Redis, and all external connections
8 CI workflows (TypeScript, ESLint, Prettier, build, audit, complexity, circular imports) running automatically
